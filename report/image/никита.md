# Введение

<p><br>Почти каждый день мы пользуемся электронными устройствами и имеем дело с файлами, с которыми всегда взаимодействуем по разному. Например, аудиофайлы мы слушаем, текстовые - читаем, а видеофайлы мы смотрим. Но как мы можем отличить одни типы файлов от других?</p></br>
Пользователи, которые не слишком углубляются в информационные технологии, обычно полагаются на иконки файлов, автоматически присваиваемые операционной системой, которые интуитивно понятны. Например, давайте посмотрим, как пользователь видит файл изображения в ОС Windows:

![Вид файла изображения в Windows](image/1.png)

Всё довольно понятно для рядового пользователя. Однако как, например, система определила, что этот файл - картинка, и что нужно подобрать именно такую иконку?
<p><br>Кроме того, встаёт другой логичный вопрос: А что, если мы лишены пользовательского интерфейса и нам, например, нужно работать с консолью в ОС Linux? Ведь там в терминале мы увидим следующее: </p></br>

![Вид файла изображения в терминале Linux](image/2.png)

Но обратим внимание на то, что после названия нашего файла стоит ".png". Это расширение имени файла. Давайте с вами разберёмся, что это такое и зачем оно нужно.

# Что такое расширение и зачем оно нужно?

Расширение имени файла - это последовательность символов, следующих после названия файла через точку и сообщающая системе и пользователю его тип. [@bigenc; @cyberleninka]

![Как выглядит структура имени файла](image/3.png)

В примере выше представлен файл с расширением "png". Но что же всё-таки нам говорят эти три буквы? Это аббревиатура от "Portable Network graphics", и сам перевод этой фразы нам говорит о том, что перед нами - графический файл, то есть изображение. Таким образом, с помощью расширения файла, система понимает 2 важные вещи, необходимые для работы с этим файлом:
<p><br>1. Как показать человеку, что это за файл. Подбор подходящей иконки - важная составляющая, поскольку для неопытных пользователей это практически единственный способ понять, что это за тип файла</p></br>
<p><br>2. Как работать с файлом. У большинства ОС есть настройки, в которых определяется, с помощью каких программ нужно открывать файлы с различными расширениями</p></br>
<p><br>Кроме того, не стоит забывать, что приложения, работающие с файлами, также нуждаются в определении его, поскольку это помогает понять, как его обработать.</p></br>

# Типы расширений

<p><br>Теперь перейдём к обзору основных существующих расширений. Они сгруппированы по типу, к которому относится файл[@microsoft_support; @habr]. </p></br>
<p><br>Для рассмотрения были выбраны расширения десяти самых частовстречающихся типов файлов, среди которых:  
1. Текстовые файлы  
2. Графические файлы  
3. Аудио-файлы  
4. Видео-файлы  
5. Архивы  
6. Документы  
7. Веб-файлы  
8. Презентации  
9. Таблицы  
10. Исполняемые файлы  
Рассмотрим раширения для каждого из этих типов:
<p><br>**Текстовые файлы**, содержащие текст:</p></br>
<p><br>txt - текстовый файл, обычно используется для хранения простой текстовой информации.</p></br>
<p><br>log - файл, содержащий логи.</p></br>

![Текстовые файлы](image/4.png)

<p><br>**Графические файлы**, содержащие изображения:</p></br>
<p><br>jpg, jpeg - файлы изображений Joint Photographic Experts Group, которые сжимают изображения с потерями для уменьшения размера файла.</p></br>
<p><br>png - файлы изображений Portable Network Graphics, которые поддерживают прозрачность и сжатие без потерь.</p></br>
<p><br>gif - файлы изображений Graphics Interchange Format, которые поддерживают анимацию и прозрачность.</p></br>
<p><br>bmp - файлы изображений Bitmap, которые сохраняют изображения без сжатия.</p></br>

![Графические файлы](image/5.jpg)

<p><br>**Музыкальные файлы**, содержащие аудио дорожки:
<p><br>mp3 - файлы MPEG-1 или MPEG-2 Audio Layer III, которые сжимают аудио с потерями для уменьшения размера файла.</p></br>
<p><br>wav - файлы Waveform Audio Format, которые сохраняют аудио без сжатия.</p></br>
<p><br>flac - файлы Free Lossless Audio Codec, которые сжимают аудио без потерь.</p></br>

![Аудио-файлы](image/6.jpg)

<p><br>**Видео-файлы**, содержащие видео и аудио дорожки:</p></br>
<p><br>mp4 - файлы MPEG-4 Part 14, которые сжимают видео и аудио с потерями для уменьшения размера файла.</p></br>
<p><br>avi - файлы Audio Video Interleave, которые поддерживают различные кодеки и форматы.</p></br>
<p><br>mkv - файлы Matroska Multimedia Container, которые поддерживают различные кодеки и форматы, а также имеют возможность "дозапысывания".</p></br>

![Видео-файлы](image/7.jpg)

<p><br>**Файлы архивов**, содержащие файлы, часто в сжатом виде:</p></br>
<p><br>zip - файлы ZIP, которые сжимают несколько файлов и папок в один файл.</p></br>
<p><br>rar - файлы RAR, которые также сжимают несколько файлов и папок в один файл.</p></br>
<p><br>7z - файлы 7-Zip, которые сжимают файлы и папки в один файл.</p></br>
<p><br>tar - файлы TAR, которые помещают файлы в архив, при этом не используя сжатие.</p></br>
<p><br>gz - файлы GNU-ZIP, которые сжимают файлы и помещают в архив</p></br>

![Архивы](image/8.jpg)

<p><br>**Файлы документов**, содержащие текст, разметку и всё содержимое документа:</p></br>
<p><br>doc, docx - файлы Microsoft Word, содержащие текстовый документ.</p></br>
<p><br>pdf - файлы Portable Document Format, которые сохраняют форматирование текста и изображений.</p></br>
<p><br>rtf - файлы Rich Text Format, которые позволяют сохранять форматирование текста.</p></br>

![Документы](image/9.png)

<p><br>**Файлы веб-страниц**, содержащие информацию для отображения сайтов:</p></br>
<p><br>html, htm: файлы HyperText Markup Language, которые содержат структуру и содержимое веб-страниц.</p></br>
<p><br>css: файлы Cascading Style Sheets, которые определяют стили для веб-страниц.</p></br>

![Веб-файлы](image/10.png)

<p><br>**Файлы презентаций**, то есть файлы, которые содержат слайды, анимации и т.д.:</p></br>
<p><br>ppt, pptx: файлы Microsoft PowerPoint, которые содержат презентации.</p></br>
<p><br>key: файлы Keynote, которые содержат презентации, созданные в программе Keynote от Apple.</p></br>

![Презентации](image/11.png)

<p><br>**Файлы электронных таблиц**, то есть файлы, которые содержат листы таблиц:</p></br>
<p><br>xls, xlsx: файлы Microsoft Excel, которые содержат электронные таблицы.</p></br>
<p><br>ods: файлы OpenDocument Spreadsheet, которые содержат электронные таблицы и поддерживаются многими программами.</p></br>

![Таблицы](image/12.png)

<p><br>**Исполняемые файлы**, то есть файлы, которые запускают приложения на компьютере:</p></br>
<p><br>exe: файлы исполняемых программ для ОС Windows</p></br>
<p><br>app: файлы исполняемых программ для MacOS</p></br>
<p><br>elf: файлы исполняемых программ для Linux, но обычно в Linux пишутся без расширения</p></br>

![Исполняемые файлы](image/13.png)

<p><br>Конечно, это далеко не все типы расширений. Важно отметить, что часть программ использует свои собственные расширения. Например, программа Adobe Photoshop использует файлы с расширением psd. 
<p><br>Ещё один важный момент заключается в том, что расширения файлов могут быть многоуровневыми. Например, мы упоминали, что файлы TAR не используют сжатие, однако могут быть дополнительно сжаты с помощью программы gzip, которая сохраняет архивы с расширением GZ. Таким образом, мы можем получить, например, файл example.tar.gz, который имеет целых два расширения, то есть имеет двухуровневое расширение.

# Выводы

<p><br>Как мы видим, существует действительно большое количество файловых расширений, которые активно используются в настоящий момент. Современный пользователь ПК, безусловно, должен знать хотя бы самые частовстречающиеся и распространённые расширения. Это поможет лучше ориентироваться в системе и испытывать меньше дискомфорта для работы с файлами. 
<p><br>И хотя может показаться, что использование расширений может быть неудобно, на самом деле большинству продвинутых пользователей довольно просто в них ориентироваться. 
<p><br>Использование расширений помогает системе определять, как работать с файлами, что делает их практически незаменимым атрибутом файлов
